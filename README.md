# Modified Double U-Net for Multi-Class Medical Image Segmentation

This repository contains a **PyTorch implementation** of the **Modified Double U-Net** architecture, specifically optimized for multi-class segmentation of medical images. This project builds upon the Double U-Net framework by introducing ensemble-based feature extraction and multi-class spatial attention gates.

## 🚀 Key Features
* **Architecture**: Dual-stacked U-Net design with an advanced ensemble encoder.
* **Backbones**: Integration of pretrained **VGG-19** (with architectural support for Xception and DenseNet feature fusion) to leverage ImageNet-learned spatial hierarchies.
* **Multi-Class Support**: Optimized for 3-class segmentation:
    * **Class 0**: Background (Normal)
    * **Class 1**: Benign
    * **Class 2**: Malignant
* **Performance Optimizations**:
    * **Automatic Mixed Precision (AMP)**: High-speed training with a reduced VRAM footprint using `torch.amp`.
    * **Persistent Workers & Memory Pinning**: Optimized DataLoader settings to eliminate CPU-GPU bottlenecks and keep GPU utilization at 100%.
    * **Combined Loss**: A robust fusion of **Cross-Entropy** and **Multi-Class Dice Loss** (with One-Hot Encoding) to effectively handle class imbalance.

---

## 🏗 Architecture Overview

The Modified Double U-Net consists of two primary subnetworks:
1.  **Network 1**: Extracts high-level features using a pretrained encoder and an **ASPP (Atrous Spatial Pyramid Pooling)** block to capture multi-scale contextual information.
2.  **Network 2**: Refines the segmentation by combining the original input image with the class-aware attention maps generated by Network 1.

**Crucial Modification**: The bridge between networks uses a **Softmax-based attention gate** (summing the probabilities of the benign and malignant classes) rather than the binary Sigmoid gate used in the original architecture. This allows the second U-Net to correctly focus on multi-class foreground pixels.

## 🛠 Prerequisites

### Hardware
* **Minimum**: NVIDIA GPU with 8GB VRAM (e.g., RTX 3060/4060).
* **Recommended**: 12GB+ VRAM for larger batch sizes and optimal worker counts.

### Environment
Ensure you have Python 3.8+ installed. Install the required dependencies:

~~~bash
pip install -r requirements.txt
~~~

## 🏋️ Training

The training script includes an **Auto-Resume Capability**—if stopped, it will automatically detect `files/checkpoint.pth` and seamlessly continue training from the last saved epoch without losing progress.

To start or resume training:

~~~bash
python train.py
~~~

**Current Training Configuration:**
* **Image Size**: 256x256
* **Batch Size**: 8
* **Optimizer**: Adam (Learning Rate: 1e-4)
* **Scheduler**: ReduceLROnPlateau 
* **Early Stopping Patience**: 50 epochs
* **Loss Function**: CrossEntropy + Multi-Class Dice Loss

## 📊 Evaluation & Inference

### Quantitative Testing
Evaluate the model on the unseen test set to generate research-grade, class-wise metrics (F1-Score, Jaccard Index/IoU, Recall, Precision). The evaluation includes tensor safety nets to prevent false metric inflation on empty classes.

~~~bash
python test.py
~~~

### Visual Prediction
Generate segmentation masks with discrete colored overlays to visually verify network performance:
* **Green Overlay**: Benign Tissue
* **Red Overlay**: Malignant Tissue

~~~bash
python predict.py
~~~
